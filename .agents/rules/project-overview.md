# Project Overview

This is a **browser-based benchmark** comparing the performance of **Vanilla JavaScript vs WebAssembly (C++)** for real-time motion detection on 4K video.

## Tech Stack

- **Frontend**: Vanilla HTML/CSS/JS (NO frameworks, NO build tools like Vite/Webpack)
- **WASM**: C++ compiled via Emscripten (`emcc`)
- **Server**: Python `http.server` (dev only)
- **Browser APIs**: Canvas 2D, Video, Performance API, requestAnimationFrame

## File Structure

| File | Role |
|---|---|
| `index.html` | UI: video upload, run/stop buttons, canvas output, metrics display |
| `js_motion.js` | Pure JavaScript implementation of motion detection algorithm |
| `motion_wasm.cpp` | C++ implementation (compiled to WASM via Emscripten) |
| `wasm_motion_bridge.js` | JS ↔ WASM bridge: memory management, function calls |
| `app.js` | Main orchestrator: video processing loop, FPS/latency measurement, UI updates |
| `build_wasm.sh` | Build script: C++ → WASM using `emcc -O3` |
| `motion_wasm.js` | Generated by Emscripten (DO NOT edit manually) |
| `motion_wasm.wasm` | Generated by Emscripten (DO NOT edit manually) |

## Motion Detection Algorithm (shared by JS and WASM)

1. **Grayscale conversion**: RGBA → luminance (`0.299R + 0.587G + 0.114B`)
2. **Box blur**: Mean filter with 7×7 kernel (radius = 3) for noise reduction
3. **Frame differencing**: Compare current blurred frame vs previous, threshold = 30

## Key Conventions

- Both JS and WASM implementations MUST use the **same algorithm** for fair comparison
- All generated files (`motion_wasm.js`, `motion_wasm.wasm`) are gitignored build artifacts
- Video files in `video4k/` are test assets, not committed to git
